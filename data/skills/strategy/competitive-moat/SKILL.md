# Competitive Moat

## Purpose
Identify and build competitive advantages: data moats, network effects, switching costs, and structural barriers.

## Steps
1. Audit current assets for existing moat potential: proprietary data, user base, brand recognition, technical IP
2. Evaluate each moat type for applicability: network effects, switching costs, data advantages, economies of scale, and brand
3. Identify the strongest moat opportunity and define what it looks like at maturity
4. Map the actions required to build the moat: what data to collect, what integrations to create, what habits to form in users
5. Assess how long the moat takes to build and how defensible it is once established
6. Analyze competitor ability to replicate: what would it cost them in time and money to match this advantage

## Output Format
A moat assessment with current state, target moat type, build plan with milestones, and a defensibility rating with competitor replication analysis.
