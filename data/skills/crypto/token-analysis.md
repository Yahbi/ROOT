# Crypto Token Analysis

Systematic framework for evaluating cryptocurrency projects before investment.

## Tokenomics Evaluation

### Supply Analysis
- **Max supply**: Is there a cap? (Bitcoin: 21M, Ethereum: no cap but net deflationary)
- **Circulating vs total**: Low float = potential future dilution as tokens unlock
- **Unlock schedule**: Plot upcoming token unlocks — large unlocks create sell pressure
- **Inflation rate**: Annual new token issuance as % of circulating supply

### Demand Drivers
- **Utility**: Is the token required to use the protocol? (gas, staking, governance)
- **Fee capture**: Does the token accrue revenue? (buyback, burn, staking yield)
- **Governance power**: Meaningful voting rights drive demand from strategic holders
- **Network effects**: More users → more utility → more demand (virtuous cycle)

### Red Flags in Tokenomics
- Team/investor allocation > 40% of total supply
- Cliff unlock events releasing > 5% of circulating supply at once
- No clear value accrual mechanism (token is purely speculative)
- Inflationary emissions without corresponding demand growth

## On-Chain Metrics

| Metric | Bullish Signal | Bearish Signal |
|--------|---------------|----------------|
| TVL trend | Growing for 3+ months | Declining despite incentives |
| Revenue/TVL | > 10% annualized | < 2% (yield tourists will leave) |
| Active users | Growing week-over-week | Declining after airdrop |
| Fee revenue | Real fees from usage | Fees only from token trading |
| Protocol-owned liquidity | > 30% of TVL | 0% (mercenary capital) |

## Developer Activity Assessment

### Metrics to Track
- Weekly GitHub commits (quality matters more than quantity)
- Unique contributors (> 20 active = healthy, < 5 = concerning)
- PR merge rate and review quality
- New feature development vs maintenance-only
- Documentation quality and update frequency

### Tools
- Electric Capital Developer Report (annual macro view)
- GitHub directly for repo-level analysis
- DeFiLlama for TVL, revenue, and user metrics

## Valuation Frameworks

### Relative Valuation
- **FDV/Revenue**: Compare to peers in same category (L1, DeFi, NFT)
- **FDV/TVL**: < 1 is potentially undervalued, > 10 is expensive
- **Market cap/Active user**: Compare adoption efficiency across protocols

### Discounted Cash Flow (for revenue-generating protocols)
1. Project protocol revenue for 5 years based on growth trajectory
2. Estimate terminal value using comparable protocol multiples
3. Discount at 30-50% (crypto risk premium is high)
4. Compare DCF value to current FDV

## Investment Checklist

- [ ] Tokenomics reviewed: supply schedule, unlock dates, value accrual
- [ ] TVL growing and revenue is from real usage (not just incentives)
- [ ] Developer activity healthy: active contributors, shipping features
- [ ] Competitive position: clear moat or differentiation vs alternatives
- [ ] Risk assessment: smart contract audits, team reputation, regulatory exposure
- [ ] Entry timing: not buying into unlocks, not chasing parabolic moves
