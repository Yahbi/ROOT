# Decision Frameworks

Apply structured thinking methods to make better decisions faster.

## Eisenhower Matrix (Urgency vs Importance)

| | Urgent | Not Urgent |
|---|--------|-----------|
| **Important** | DO: Crises, deadlines, critical bugs | SCHEDULE: Strategy, learning, relationships, health |
| **Not Important** | DELEGATE: Most emails, some meetings, interruptions | ELIMINATE: Social media, busywork, unnecessary meetings |

### Key Insight
- Most people spend 80% of time in Urgent quadrants
- Highest-leverage work lives in Important + Not Urgent (schedule proactively)
- If everything is urgent, nothing is — reassess priorities

## OODA Loop (Observe, Orient, Decide, Act)

### Phases
1. **Observe**: Gather raw data about the situation (metrics, signals, feedback)
2. **Orient**: Analyze and synthesize — what does this mean? (context, mental models, experience)
3. **Decide**: Select a course of action from available options
4. **Act**: Execute the decision, then loop back to Observe

### When to Use
- Fast-changing environments (markets, competitive situations, incident response)
- The goal: cycle through OODA faster than your competition
- Speed of decision matters more than perfection — a good decision now beats a perfect decision later

## First Principles Thinking

### Method
1. **Identify the problem**: What are you actually trying to achieve?
2. **Break it down**: What are the fundamental truths (not assumptions)?
3. **Challenge assumptions**: "Everyone does it this way" is not a first principle
4. **Reason up**: Build a solution from the fundamentals

### Example
- Assumption: "We need to hire more engineers to ship faster"
- First principles: What's actually slowing us down? (meetings, tech debt, unclear specs?)
- Solution might be: reduce meetings + fix deploy pipeline instead of hiring

## Other Useful Frameworks

### 10/10/10 Rule
- How will I feel about this decision in 10 minutes? 10 months? 10 years?
- Reduces emotional decision-making by forcing long-term perspective

### Reversibility Test
- **Reversible decisions (two-way door)**: Decide quickly, correct later if wrong
- **Irreversible decisions (one-way door)**: Invest time in analysis, gather more data
- Most decisions are reversible — people over-analyze them

### Pre-Mortem
1. Assume the project failed. Write the headline: "Project X failed because..."
2. List all the reasons it could have failed
3. Now prevent the most likely failure modes before they happen
4. More effective than post-mortem (prevention > cure)

### Weighted Decision Matrix
1. List options as columns, criteria as rows
2. Weight each criterion by importance (sum to 100%)
3. Score each option against each criterion (1-5)
4. Multiply and sum: highest weighted score wins
5. Use when multiple factors compete and gut feeling isn't enough

## Decision Quality Checklist

- [ ] Problem clearly defined (what am I actually deciding?)
- [ ] Options identified (minimum 3 — avoid binary thinking)
- [ ] Key criteria listed and weighted
- [ ] Reversibility assessed (one-way door vs two-way door)
- [ ] Second-order effects considered (what happens after what happens?)
- [ ] Decision documented with rationale (for future learning)
