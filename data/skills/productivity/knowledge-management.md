# Knowledge Management

Capture, organize, and retrieve knowledge effectively for continuous learning and decision-making.

## Note-Taking Systems

### Zettelkasten Method
1. **Fleeting notes**: Quick captures during the day (thoughts, ideas, quotes)
2. **Literature notes**: Summarize what you read in your own words (one idea per note)
3. **Permanent notes**: Refined, standalone ideas linked to other notes
4. Key rule: each note should make sense without context — write for your future self
5. Link notes bidirectionally — the network of connections creates emergent insights

### PARA Method (Tiago Forte)
| Category | Purpose | Examples |
|----------|---------|---------|
| **P**rojects | Active work with deadlines | "Launch product X", "Write proposal" |
| **A**reas | Ongoing responsibilities | "Health", "Finances", "Team management" |
| **R**esources | Topics of interest | "Machine learning", "Investing", "Design" |
| **A**rchives | Inactive items | Completed projects, old resources |

### Progressive Summarization
1. Save the full article/chapter (Layer 1)
2. Bold the important passages (Layer 2)
3. Highlight the critical sentences within bolded text (Layer 3)
4. Write a 1-sentence summary at the top (Layer 4)
5. Each layer makes the note faster to re-read and easier to use

## Spaced Repetition

### How It Works
- Review information at increasing intervals: 1 day, 3 days, 7 days, 14 days, 30 days
- Material you recall easily gets longer intervals (less review)
- Material you forget gets shorter intervals (more review)
- 15-20 minutes daily maintains large volumes of knowledge

### Effective Flashcard Design
- One concept per card (atomic)
- Use cloze deletions: "The Kelly criterion formula is ___"
- Include context: "In portfolio optimization, the Sharpe ratio measures ___"
- Add images, code snippets, or diagrams when relevant
- Write cards from your notes, not copied verbatim from sources

## Knowledge Retrieval

### Search-Friendly Practices
- Tag notes with 2-5 keywords (not too many, not too few)
- Use consistent naming conventions: `YYYY-MM-DD-topic-name`
- Write descriptive titles (searchable) not clever titles
- Cross-reference related notes with explicit links

### Weekly Review
1. Process inbox: convert fleeting notes to permanent notes
2. Update project notes with progress and decisions
3. Review upcoming tasks and deadlines
4. Identify knowledge gaps: what do I need to learn this week?
5. Archive completed project notes

## Anti-Patterns

- Collecting without processing (hoarding, not learning)
- Over-organizing (spending more time on the system than using it)
- Perfect note syndrome (notes don't need to be polished — they need to be useful)
- Not reviewing (knowledge decays without retrieval practice)
