# System Prompt Design

## Purpose
Design effective system prompts that give LLMs clear instructions, constraints, and context to produce consistent, high-quality outputs aligned with the intended use case.

## Steps
1. Define the role, persona, and expertise level the LLM should adopt
2. Specify the task objective and desired output format explicitly
3. Set constraints and boundaries (what to do, what to avoid, tone, length)
4. Provide examples of ideal inputs and outputs (few-shot examples)
5. Add edge case handling instructions and fallback behaviors
6. Structure the prompt with clear sections using headers or delimiters
7. Test with diverse inputs, iterate on wording, and measure output quality

## Output Format
A complete system prompt document with role definition, task instructions, constraints, examples, and edge case handling, ready to be used in an LLM configuration.
